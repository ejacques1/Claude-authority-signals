# Claude Authority Signals

**Companion code repository for:**

Jacques, E., Datuowei, E., Quaye, E., Basch, C. H., Chatterjee, A., & Davis, J. (2026). Authority signals in Claude AI health citations: A descriptive analysis using the Authority Signals Framework.

**Pipeline developed by Dr. Erin Jacques**

---

## Overview

This repository contains the data collection and analysis pipeline used to examine authority signals in 10,038 health citations generated by Anthropic's Claude AI (Sonnet 4.5) in response to 3,172 consumer health questions from the HealthSearchQA dataset. The study applies the Authority Signals Framework (Jacques et al., 2026) to analyze 10 authority signals across four domains for a focused sample of 542 sources.

Data collection was conducted in February 2026 using default API parameters (temperature 1.0).

## Phase 1: Citation Extraction

In February 2026, 3,172 consumer health questions from HealthSearchQA were stored in a Supabase PostgreSQL database. The Claude Sonnet 4.5 API was called for each question to generate source citations, yielding 12,006 URLs. Those URLs were scraped by Apify to retrieve source names and organizational types, with results written to Google Sheets via an automated pipeline deployed on Vercel.

After excluding 64 questions with no citations and 33 questions with non-health citations, the final dataset comprised 3,075 questions and 10,038 citations classified across 7 organization types.

**Files:**
- `api/extract-citations.js` — Main cron endpoint. Processes batches of 20 questions, calls the Claude API, extracts cited URLs, and writes results to Google Sheets. Progress is tracked in Supabase.
- `api/trigger.js` — Manual trigger to run extraction batches on demand.
- `scripts/load-questions.js` — One-time setup script to load HealthSearchQA questions into Supabase.
- `scripts/sync-urls-to-supabase.js` — Syncs extracted URLs from Google Sheets to Supabase for Phase 3 coding.

## Phase 2: Disproportionate Stratified Sampling

A disproportionate stratified sampling strategy was used to construct a focused analysis sample. A Fisher-Yates shuffling algorithm implemented in Google Apps Script selected a 5% random sample from organization types with more than 100 citations. The full census was used for types with fewer than 100 citations. The full census of Commercial Health Information was also included to enable detailed comparison. After removing 296 broken URLs, the focused analysis sample was n = 542.

**Note:** The Fisher-Yates shuffling script was executed in Google Sheets and is not preserved in this repository. The sampling procedure is fully described in the manuscript.

---

## Phase 3: Authority Signal Coding

The 542 focused sample sources were analyzed across 10 authority signals in four domains. An automated pipeline scraped each URL's content using Apify, packaged the HTML with the operational definitions, and sent them to Claude Sonnet 4.5 for coding. Results were written to Google Sheets. Domain Authority, Page Authority, and Spam Score were collected via the Moz API. Three human coders reviewed all 542 sources across all 10 variables to verify automated coding accuracy.

**Files:**
- `api/code-authority-signals.js` — Main coding endpoint. Contains the complete operational definitions for all authority signal variables embedded in the coding prompt. Scrapes webpage content via Apify, sends content to Claude for analysis, and writes coded results to Google Sheets.
- `api/trigger-coding.js` — Manual trigger to run coding batches on demand.

## Operational Definitions

The Authority Signals Framework examines 10 signals across four domains:

1. **Who wrote it?** (Author Credentials) — Author attribution level
2. **Who published it?** (Institutional Affiliation) — Organization type (7 categories)
3. **How was it vetted?** (Quality Assurance) — References cited, medical review statement, content recency
4. **How does AI find it?** (Digital Authority) — Page authority, domain authority, spam score, content length, schema markup

Complete operational definitions with coding instructions are embedded in `api/code-authority-signals.js` and are also archived on Zenodo: [insert DOI]
