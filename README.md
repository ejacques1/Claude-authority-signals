# Claude Authority Signals

**Companion code repository for:**

Jacques, E., Datuowei, E., Quaye, E., Basch, C. H., Chatterjee, A., & Davis, J. (2026). Authority signals in Claude AI health citations: A descriptive analysis using the Authority Signals Framework.

**Pipeline developed by Dr. Erin Jacques**

---

## Overview

This repository contains the data collection and analysis pipeline used to examine authority signals in 10,038 health citations generated by Anthropic's Claude AI (Sonnet 4.5) in response to 3,172 consumer health questions from the HealthSearchQA dataset. The study applies the Authority Signals Framework (Jacques et al., 2026) to analyze 10 authority signals across four domains for a focused sample of 542 sources.

## Phase 1: Citation extraction

In February 2026, 3,172 consumer health questions from HealthSearchQA were stored in a Supabase PostgreSQL database. The Claude Sonnet 4.5 API was called for each question to generate source citations, yielding 12,006 URLs. Those URLs were scraped by Apify to retrieve source names and organizational types, with results written to Google Sheets via an automated pipeline deployed on Vercel.

After excluding 64 questions with no citations and 33 questions with non-health citations, the final dataset comprised 3,075 questions and 10,038 citations classified across 7 organization types.

**Code included:** `api/extract-citations.js`, `api/trigger.js`, `scripts/load-questions.js`, `supabase-schema.sql`, `vercel.json`

## Phase 2: Disproportionate stratified sampling

A disproportionate stratified sampling strategy was used to construct a focused analysis sample. A Fisher-Yates shuffling algorithm implemented in Google Apps Script selected a 5% random sample from organization types with more than 100 citations. The full census was used for types with fewer than 100 citations. The full census of Commercial Health Information was also included as the organization type of interest. After removing 296 broken URLs, the focused analysis sample was n = 542.

**Code note:** The Fisher-Yates shuffling script was executed in Google Sheets and is not preserved in this repository. The sampling procedure is fully described in the manuscript.

---

## Phase 3: Authority signal coding

The 542 focused sample sources were analyzed across 10 authority signals in four domains. An automated pipeline packaged HTML source code from each URL with the 10 authority signal operational definitions and sent them to Claude Sonnet 4.5 for coding. Results were written to Google Sheets. Domain Authority, Page Authority, and Spam Score were collected via the Moz API. Three human coders reviewed all 542 sources across all 10 variables to verify automated coding accuracy.

**Code included:** Two Google Apps Script functions used during Phase 3 coding are preserved in `google-apps-scripts/`:
- `checkSchema.gs` — Detects schema markup presence and type (JSON-LD, Microdata, RDFa)
- `checkAuthor.gs` — Detects author names and professional credentials from HTML source

**Code note:** The full Node.js authority signal coding pipeline was deployed on Vercel and is not fully preserved. Complete operational definitions for all 10 authority signals are included in `operational-definitions/`.

## Operational definitions

The Authority Signals Framework examines 10 signals across four domains:

1. **Who wrote it?** (Author Credentials) — Author attribution level
2. **Who published it?** (Institutional Affiliation) — Organization type (7 categories)
3. **How was it vetted?** (Quality Assurance) — References cited, medical review statement, content recency
4. **How does AI find it?** (Digital Authority) — Page authority, domain authority, spam score, content length, schema markup

Complete operational definitions with coding instructions are available in `operational-definitions/`.
